Git的特点

  • 最优的存储能力
  • 非凡的性能
  • 开源的
  • 很容易做备份
  • 支持离线操作
  • 很容易定制工作流程

安装地址:https://git-scm.com/book/zh/v2.%E8%B5%B7%E6%AD%A5-%E5%AE%89%E8%A3%85-Git

查看是否安装成功:git –version

Git最小配置

配置user信息:即配置user.name和user.email

git config —-global user.name = ‘your name’
git config —-global user.email ‘your email’

git config共有三个作用域

缺省等同于local

git config --local 只对某个仓库有效
git config --global global对当前用户所有仓库有效
gut config --system system对系统所有登录的用户有效

显示config的配置，加–list

git config --list --local
git config --list --global
git config --list --system

配置优先级local > global > system

创建git仓库

1、把已有的项目代码纳入git管理

cd 项目所在文件夹
git init

2、新建的项目直接用git管理

cd 某个文件夹
git init projectname
cd projectname

提交代码

git add name 添加到暂存区里,可以一次提交多个文件，可以用-u添加所有文件
git commit -m'Add readme'

git status查看git状态 git log查看提交记录

文件重命名

方式一: 工作区重命名(mv name new_name) git add git rm 方式二：git mv name new_name

git log详解

简单看一下列表：git log –oneline 最近四条:git log -n4 查看所有分支的版本历史:git log –all 图形化地查看多个分支演进:git log –all –graph git branch -v可以查看有多少分支

通过gitk可以使用图形界面工具来看版本历史

git有三种文件类型commit、tree(文件夹)、blob(文件)

删除:git branch -d branch_name、git branch -D branch_name

修改commit最近一次提交的message:git commit –amend 修改老的commit的message：git rebase -i id号 合并多个commit:git rebase -i id号

比较暂存区和head的区别:git diff –cached 比较工作区和暂存区的区别:git diff

把暂存区恢复到和head一样:git reset HEAD 把工作区恢复到跟暂存区一样:git checkout – filename 取消暂存区部分文件的修改:git reset HEAD – filename 消除最近的几次提交

消除最近的几次提交:git reset –hard 指定位置的id

开发中临时加塞紧急任务处理:

git stash//存起来，工作区清除 git stash list git stash applay//恢复
